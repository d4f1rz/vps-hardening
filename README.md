# 🔐 VPS Hardening Script

Интерактивный Bash-скрипт для усиления безопасности VPS на Ubuntu/Debian после первого входа под `root`. Скрипт полностью автоматизирует hardening-пайплайн: обновления, создание нового admin-пользователя, генерацию SSH-ключей, настройку SSH/UFW/Fail2ban и вывод финального отчета с данными подключения.

## ⚡ Быстрый старт (1 команда)

```bash
curl -fsSL https://raw.githubusercontent.com/d4f1rz/vps-hardening/master/vps_hardening.sh | sudo bash
```

## ⚡ Быстрый старт (3 команды)

```bash
wget -O vps_hardening.sh https://raw.githubusercontent.com/d4f1rz/vps-hardening/master/vps_hardening.sh
chmod +x vps_hardening.sh
sudo bash vps_hardening.sh
```

## 📋 Что делает скрипт

| Шаг | Описание | Интерактивный? |
|-----|----------|----------------|
| 0 | Предпроверки: root, ОС, старт | Да |
| 0.1 | Выбор модели IP-доступа: `all` или `selected` | Да |
| 1 | Обновление системы (`apt update`, `apt upgrade -y`) | Нет |
| 2 | Создание пользователя, авто-генерация пароля, удаление старых пользователей (кроме `root` и нового) | Частично |
| 3 | Полная регенерация SSH-ключей для нового пользователя и перезапись `authorized_keys` | Нет |
| 4 | Усиление SSH-конфига и проверка `sshd -t` | Частично |
| 5 | Настройка UFW: открывается только SSH-порт по выбранной IP-модели | Нет |
| 6 | Настройка Fail2ban с `ignoreip` текущего клиента и усиленным баном | Нет |
| 7 | Перезапуск SSH и финальная проверка | Нет |
| 8 | Большой итоговый отчет с командами доступа и ключами | Нет |

## 📖 Полная инструкция по использованию

### Шаг 0: Подготовка — что нужно заранее

- Доступ к серверу под `root` (SSH или консоль провайдера)
- Рекомендуется snapshot VPS перед запуском
- Понимание, что скрипт удалит старых пользовательских аккаунтов (кроме `root` и нового)

### Запуск скрипта

Одной строкой (скачать и сразу запустить):

```bash
curl -fsSL https://raw.githubusercontent.com/d4f1rz/vps-hardening/master/vps_hardening.sh | sudo bash
```

Классический вариант (скачать, проверить, запустить):

```bash
wget -O vps_hardening.sh https://raw.githubusercontent.com/d4f1rz/vps-hardening/master/vps_hardening.sh
chmod +x vps_hardening.sh
sudo bash vps_hardening.sh
```

### Что изменилось в новом автоматическом режиме

- Больше нет подтверждения `Y/n` перед каждым шагом.
- Больше нет пауз `Нажмите Enter` после каждого шага.
- Перед шагом показывается красивый заголовок с описанием, что и зачем будет сделано.
- Подтверждение остается только на запуск всего сценария и при критической ошибке.

### Выбор модели доступа по IP (важно для VPS1 ↔ VPS2)

Скрипт предлагает 2 режима:

1. `all` — SSH вход разрешен с любых IP.
2. `selected` — SSH вход разрешен только с указанных IP/CIDR.

Для схемы с `Marzban (main)` и `Marzban-node` обычно рекомендуется `selected`, например:

- IP `VPS1` (main)
- Ваш доверенный домашний/рабочий IP

### Что происходит во время выполнения

**Шаг 1. Обновление системы**

- Выполняется `apt update`, затем `apt upgrade -y`.
- В консоли показывается однострочный прогресс/статус текущей команды.

```text
[RUN] Обновление списка пакетов (apt update) /
[OK] Обновление списка пакетов (apt update)
```

**Шаг 2. Пользователь и пароль**

- Вы вводите только имя нового пользователя.
- Пароль генерируется автоматически (16 символов: буквы, цифры, спецсимволы).
- Старые пользователи удаляются автоматически (кроме `root` и нового).
- Новый пользователь добавляется в `sudo`.

```text
╔══════════════════════════════════════════════╗
║   ДАННЫЕ ПОЛЬЗОВАТЕЛЯ (СОХРАНИТЕ ИХ)        ║
╠══════════════════════════════════════════════╣
║ User:     admin                             ║
║ Password: xY7#...                           ║
╚══════════════════════════════════════════════╝
```

**Шаг 3. SSH-ключи (полностью автоматически)**

- Старые `authorized_keys` удаляются.
- На сервере генерируется новый `ed25519` ключ:

```bash
ssh-keygen -t ed25519 -C "example@example.com"
cat ~/.ssh/id_ed25519.pub
```

- Passphrase генерируется автоматически (128 символов).
- Ключ автоматически прописывается в `authorized_keys` и выставляются права.
- В финале скрипт выводит `private key`, `public key`, `passphrase`.

**Шаг 4. Усиление SSH-доступа**

- Запрос нового порта с подсказкой диапазона: `1024-65535` (Enter = `2222`).
- Применяются параметры hardening:
  - `PermitRootLogin no`
  - `PasswordAuthentication no`
  - `PubkeyAuthentication yes`
  - `MaxAuthTries 3`
  - и др.
- Выводится команда подключения:

```bash
ssh <user>@<IP> -p <port>
```

**Шаг 5. UFW**

- Политики: `deny incoming`, `allow outgoing`.
- Открывается только SSH-порт, выбранный на шаге 4.
- Для режима `selected` доступ разрешается только с введенных IP/CIDR.

**Шаг 6. Fail2ban**

Создается `jail.local` с параметрами:

- `ignoreip`: `127.0.0.1`, `::1` и IP вашего текущего подключения
- `bantime = 24h`
- `findtime = 300`
- `maxretry = 3`
- порт автоматически подставляется из SSH-порта
- включен инкрементальный бан с ростом до постоянного

**Шаг 7. Проверка SSH**

- Перезапуск SSH-сервиса
- Проверка статуса сервиса

**Шаг 8. Финальный отчет**

- Большая итоговая таблица:
  - `IP:port`
  - команда подключения
  - `user/password`
  - `private key/public key/passphrase`
  - путь к логу

### После завершения: как подключиться

```bash
ssh -p <ВАШ_ПОРТ> <ВАШ_ПОЛЬЗОВАТЕЛЬ>@<IP_СЕРВЕРА>
```

## 🚨 Устранение неполадок (Troubleshooting)

| Проблема | Вероятная причина | Решение |
|----------|------------------|---------|
| Не удается войти по SSH после запуска | Неверный порт или IP-ограничение UFW (`selected`) | Проверить `ufw status verbose`, открыть доступ через консоль провайдера |
| `Permission denied (publickey)` | Используется не тот private key или passphrase | Убедиться, что используется ключ из финального отчета |
| Пользователь исчез | Скрипт удалил старых пользователей по политике hardening | Входить новым пользователем из отчета |
| Fail2ban банит легитимный IP | Слишком много ошибок входа | Разбанить: `fail2ban-client set sshd unbanip <IP>` |
| Ошибка в SSH-конфиге | Не прошла валидация `sshd -t` | Восстановить `/etc/ssh/sshd_config.bak` |

## 🛡️ Что защищает каждый шаг

- **Обновления пакетов**: закрывают известные CVE.
- **Новый отдельный sudo-user**: снижает риск прямой компрометации root-сессии.
- **Удаление старых пользователей**: убирает забытые учетные записи.
- **Автогенерация новых SSH-ключей**: убирает риск старых/утекших ключей.
- **SSH hardening + новый порт**: уменьшает шумовые атаки и brute-force.
- **UFW с IP-моделью доступа**: дает ACL-контроль, полезный для связки VPS1↔VPS2.
- **Fail2ban с ростом банов**: блокирует перебор и рецидивы атак.

## ⚠️ Важные предупреждения

- Скрипт выводит чувствительные данные (`password`, `private key`, `passphrase`) прямо в консоль: сразу сохраните их в безопасном менеджере секретов.
- Не закрывайте текущую сессию, пока не проверите новый вход.
- При режиме `selected` обязательно добавьте ваш рабочий IP, иначе можно потерять доступ.
- Перед запуском рекомендуется snapshot VPS.
