# 🔐 VPS Hardening Script

Интерактивный Bash-скрипт для быстрой базовой защиты нового VPS на Ubuntu/Debian сразу после первого входа под `root`. Скрипт автоматизирует ключевые шаги hardening: создание отдельного sudo-пользователя, переход на SSH-ключи, усиление SSH, настройку UFW и Fail2ban. Подходит как для ручного запуска, так и для повторного безопасного прогона с учетом идемпотентности.

## ⚡ Быстрый старт (3 команды)

```bash
wget -O vps_hardening.sh https://raw.githubusercontent.com/d4f1rz/vps-hardening/master/vps_hardening.sh
chmod +x vps_hardening.sh
sudo bash vps_hardening.sh
```

## 📋 Что делает скрипт

| Шаг | Описание | Интерактивный? |
|-----|----------|----------------|
| 0 | Предпроверки: root, ОС, подтверждение старта | Да |
| 1 | Обновление системы (`apt update && apt upgrade -y`) | Да (подтверждение шага) |
| 2 | Создание/переиспользование sudo-пользователя | Да |

| 6 | Установка и настройка Fail2ban | Да (подтверждение шага) |
| 7 | Перезапуск SSH и финальная проверка | Да (подтверждение шага) |
| 8 | Итоговый отчёт с командой подключения и логом | Нет |

## 📖 Полная инструкция по использованию

### Шаг 0: Подготовка — что нужно иметь заранее

- Доступ к серверу по SSH (`root` + пароль из панели хостинга)
- SSH-клиент: Termius (рекомендуется), PuTTY (Windows) или Terminal (Linux/macOS)
- Публичный SSH-ключ (см. ниже)
- Желательно: снимок/backup VPS перед запуском

### Как сгенерировать SSH-ключ (подробная инструкция)

**Linux / macOS:**

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
cat ~/.ssh/id_ed25519.pub   # скопировать этот вывод
```

**Windows (PowerShell):**

```powershell
ssh-keygen -t ed25519 -C "your_email@example.com"
Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub
```

**Через Termius (GUI):**

1. Откройте `Keychain`.
2. Нажмите `Add Key`.
3. Выберите `Generate`.
4. Алгоритм: `ED25519`.
5. Скопируйте поле `Public Key`.

### Подключение к серверу первый раз

1. Откройте Termius и создайте новый Host.
2. Введите IP сервера.
3. Порт: `22`.
4. Username: `root`.
5. Password: пароль от панели хостинга.
6. Подключитесь и выполните запуск скрипта.

### Запуск скрипта

```bash
wget -O vps_hardening.sh <URL>
chmod +x vps_hardening.sh
sudo bash vps_hardening.sh
```

### Что происходит во время выполнения

Ниже кратко описано, что вы увидите на каждом этапе.

**Шаг 1. Обновление системы**

```text
[INFO] Команда: apt update
[OK] Выполнено
[INFO] Команда: DEBIAN_FRONTEND=noninteractive apt upgrade -y
[OK] Выполнено
```

**Шаг 2. Создание sudo-пользователя**

```text
Введите имя нового пользователя:
Введите пароль (минимум 12 символов):
Подтвердите пароль:
[OK] Пользователь добавлен в sudo
```

**Шаг 3. SSH-ключи**

```text
Вставьте публичный SSH-ключ:
[OK] Публичный ключ добавлен.
[INFO] Команда: chmod 700 /home/<user>/.ssh
```

**Шаг 4. SSH daemon**

```text
Введите новый SSH-порт [2222]:
[INFO] Настройка /etc/ssh/sshd_config
[INFO] Команда: sshd -t
[OK] Выполнено
[WARN] Не закрывайте текущую сессию!
```

**Шаг 5. UFW**

```text
[INFO] Команда: ufw default deny incoming
[INFO] Команда: ufw allow <port>/tcp
Открыть дополнительные TCP-порты? (8080,3000)
[INFO] Команда: ufw status verbose
```

**Шаг 6. Fail2ban**

```text
[INFO] Команда: apt install -y fail2ban
[INFO] Создание /etc/fail2ban/jail.local
[INFO] Команда: fail2ban-client status sshd
```

**Шаг 7. Финальная проверка**

```text
[INFO] Команда: systemctl restart sshd
[INFO] Команда: systemctl status sshd --no-pager
```

После завершения скрипт покажет итоговый блок с важными данными подключения.

### После завершения: как подключиться

```bash
ssh -p <ВАШ_ПОРТ> <ВАШ_ПОЛЬЗОВАТЕЛЬ>@<IP_СЕРВЕРА>
```

Примечание для Termius: откройте Host и обновите `Username`, `Port`, затем добавьте приватный ключ в настройках аутентификации.

## 🚨 Устранение неполадок (Troubleshooting)

| Проблема | Вероятная причина | Решение |
|----------|------------------|---------|
| Не могу подключиться после запуска | Неверный порт в SSH-клиенте | Проверить порт: `ss -tlnp | grep sshd` |
| `Permission denied (publickey)` | Ключ не добавлен или вставлен с ошибкой | Проверить `~/.ssh/authorized_keys` и права `700/600` |
| UFW заблокировал доступ | Не был разрешён SSH-порт перед `ufw enable` | Подключиться через VNC/Serial-консоль хостинга и открыть порт вручную |
| Fail2ban забанил меня | Много неудачных попыток входа | Разбанить IP: `fail2ban-client set sshd unbanip <IP>` |
| `sshd -t` выдаёт ошибку | Неправильный синтаксис в `sshd_config` | Восстановить бэкап: `/etc/ssh/sshd_config.bak` |
| Не работает `sudo` у нового пользователя | Пользователь не в группе `sudo` | Выполнить: `usermod -aG sudo <user>` |

## 🛡️ Что именно защищает каждый шаг

- **Обновление пакетов**: закрывает известные CVE-уязвимости в уже установленных пакетах.
- **Отдельный пользователь**: реализует принцип минимальных привилегий и снижает риски при компрометации.
- **SSH-ключ вместо пароля**: делает брутфорс паролей практически бесполезным.
- **Смена порта SSH**: отсекает значительную часть автоматических сканеров и шумовых атак.
- **UFW**: блокирует входящие подключения по умолчанию и оставляет только явно разрешённые сервисы.
- **Fail2ban**: автоматически банит IP-адреса при серии неудачных попыток входа.

## ⚠️ Важные предупреждения

- Не закрывайте текущую SSH-сессию, пока не проверите вход новым пользователем и новым портом.
- Обязательно сохраните итоговый блок с данными подключения.
- Перед запуском рекомендуется сделать snapshot VPS.
- Если используете `--dry-run`, реальные изменения не применяются.
